# 🤝 Contributing to ucash.eth

Thank you for your interest in contributing to the ucash.eth ENS repository!

---

## 📋 How to Contribute

### Reporting Issues

If you find an issue with the repository:
1. Check existing issues first
2. Create a new issue with:
   - Clear title
   - Detailed description
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Improvements

We welcome suggestions for:
- New subdomain documentation
- Additional verification methods
- Integration examples
- Security improvements
- Documentation enhancements

### Pull Requests

**Before creating a PR**:
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests if applicable
5. Update documentation
6. Submit your PR

**PR Requirements**:
- Clear description of changes
- Link to related issues
- No breaking changes without discussion
- Follow existing code style

---

## 📝 Documentation Updates

### Adding New Subdomains

When documenting new subdomains, include:
- Subdomain name
- Type (CNAME/ADDR)
- Target address
- Purpose
- Status
- Date added

Example:
```markdown
| Subdomain | Type | Target | Purpose | Status |
|-----------|------|--------|---------|--------|
| `newservice.ucash.eth` | ADDR | `0x...` | New service | 🟢 Active |
```

### Updating Addresses

When updating addresses:
1. Mark old address as deprecated
2. Add new address with verification date
3. Update all references
4. Add migration note if applicable

## 🔒 Security Contributions

### Responsible Disclosure

**Do NOT**:
- Create public issues about security vulnerabilities
- Disclose vulnerabilities before fix is deployed
- Exploit vulnerabilities

**DO**:
- Email security@ucash.eth
- Include detailed description
- Allow time for fix
- Follow responsible disclosure

### Bug Bounty

Security researchers may be eligible for bounties:
- See SECURITY.md for details
- Report via security@ucash.eth
- Allow responsible disclosure timeline

## 🎨 Style Guide

### Markdown Formatting
- Use proper heading hierarchy
- Include tables for structured data
- Add emojis for visual clarity
- Use code blocks for technical content
- Include examples where helpful

### Address Formatting
```
Primary: [0x1234...]
Description: What this address is for
ENS: [optional ENS]
Verified: [✅/❌]
```

### Status Indicators
- 🟢 Active
- 🟡 Testing/Coming Soon
- 🔴 Deprecated
- ⚪ Planned
